# Contributing to Guptrix

Thank you for your interest in contributing to Guptrix! As a security-focused open-source project, we welcome contributions that improve the privacy, performance, and usability of the platform.

## How to Contribute

1. **Fork the Repository**: Create your own fork of the project.
2. **Create a Branch**: Make a new branch for your feature or bug fix (`git checkout -b feature/your-feature-name`).
3. **Commit Changes**: Make sure your commits are descriptive and logical.
4. **Push to Fork**: Push your branch to your forked repository.
5. **Open a Pull Request**: Submit a PR to the main repository for review.

## Code Guidelines

- **Security First**: Do not introduce any code that compromises the end-to-end encryption layer.
- **No Secrets**: Never commit `.env` files, API keys, keystores, or passwords.
- **Formatting**: Adhere to the existing code style (Prettier for TypeScript, standard Kotlin formatting for Android).

## Reporting Bugs

Please use the GitHub Issue Tracker to report bugs. Include steps to reproduce, device information, and relevant error logs (sanitized of sensitive data).
